# Office Survival Kit — Documentation Roadmap

**Purpose:** Complete list of documents needed to take this product from concept to launch, organized by phase and priority.

**Version:** v3
**Last Updated:** March 11, 2026

**Architecture note (March 10, 2026):** Each document in this project has one specific job. No specs are duplicated across documents. If you need to know something, consult the authority document for that topic — see the Cross-Reference Map below.

**Cascade note (March 11, 2026):** The interior was redesigned from a vertical 3-tray stack to a 2-zone side-by-side layout with hidden mug compartment. Box dimensions changed from 13" × 8" × 6.5" to 14" × 11.5" × 5.0" (all working estimates). This triggered a full correction cascade across all product definition and production documents. See the Authority Block for the complete delta.

## Phase 1: Foundation Documents

*Create these FIRST — they guide everything else.*

---

### 1.1 Project Brief

**Purpose:** Single source of truth for what this project is.

**Contents:**
- Project name
- One-sentence description
- Target customer (buyer AND recipient)
- Target price point
- Target launch date
- Success metrics (units, revenue, margin)
- Key constraints (budget, timeline, brand fit)
- Stakeholders and roles
- Links to all other documents

**Status:** ⬜ Not started

---

### 1.2 Pre-Mortem Analysis

**Purpose:** Anticipate and prevent failure.

**Contents:**
- Failure scenarios
- Root causes
- Risk assessment
- Critical vulnerabilities
- Early warning signals

**Status:** ✅ Complete

---

### 1.3 Action Plan

**Purpose:** Prioritized steps to execute.

**Contents:**
- Critical actions (immediate)
- High priority actions (weeks 2-4)
- Medium priority actions (weeks 5-8)
- Decision points / go-no-go gates
- Checklist format for tracking

**Status:** ✅ Complete — ⚠️ may need refresh after layout redesign

---

### 1.4 Risk Register

**Purpose:** Living document to track identified risks.

**Contents:**
- Risk name
- Likelihood (H/M/L)
- Impact (H/M/L)
- Status (Red/Yellow/Green)
- Mitigation strategy
- Owner
- Last updated

**Format:** Spreadsheet recommended

**Status:** ⬜ Not started

---

### 1.5 Goals & Success Metrics

**Purpose:** Define what success looks like, measurably.

**Contents:**
- Business goals (revenue, margin, units)
- Customer goals (buyer confidence, recipient delight)
- Quality goals (reviews, return rate)
- Specific targets with numbers

**Status:** ✅ Complete

---

### 1.6 Item Ideation (Decisions Log)

**Purpose:** Historical record of creative and packaging decisions — what was considered, what was chosen, and why.

**Contents:**
- Item shortlist evolution
- Packaging concept development
- Tray groupings rationale
- Inside lid design options
- Key creative decisions and rationale
- Open questions (historical)

**Status:** ✅ Complete — historical reference, not a living spec. Some packaging decisions (3-tray stack, binder rings, D-rings) are now superseded by the layout redesign. Retained as historical context.

## Phase 2: Product Definition Documents

*Create these before sourcing meetings.*

---

### 2.1 Product Spec — Kit Overview

**Purpose:** Define the complete product at a high level.

**Contents:**
- Kit name and tagline
- Number of items
- Target COGS (cost of goods sold)
- Target retail price
- Target margin
- Container/packaging type
- Unboxing experience description

**Status:** ⬜ Not started

---

### 2.2 Item Specifications

**Purpose:** Detailed spec for each item in the kit — what sourcing needs.

**One page per item containing:**
- Item name (internal + product name)
- What it is (physical description)
- Dimensions
- Material
- Customization required (print, color, shape)
- Reference images / inspiration
- Copy that appears on item
- Quantity per kit
- Notes / special requirements

- ✅ Lifer Ink (temporary tattoos) — note: was "Achievement Tats", renamed

- ✅ Contraband Cry Rags (tissues)
- ✅ Solitary (ear plugs)
- ✅ Ankle Monitor (wristband)
- ✅ Evidence Eraser (novelty eraser)
- ✅ Mug Shot (enamel mug)
- ✅ Rap Sheet (notepad)
- ✅ Corporate Parole Passport (booklet)
- ✅ Yard Pass (novelty card)

**Status:** ✅ Complete (12 items, updated March 11, 2026 — tray assignments and placement notes updated for 2-zone layout; Passport and Yard Pass moved to mug compartment)

---

### 2.3 Copy Document

**Purpose:** All written content in one place.

**Contents:**
- Kit name and tagline options
- Kit description (short — 1-2 sentences)
- Kit description (long — for Amazon listing)
- Per-item copy:
  - Item name
  - Tagline (short — for packaging)
  - Description (medium — for listing)
  - Copy that appears ON the item itself
- Packaging copy (front, back, inside)
- Any inserts (cards, instructions, etc.)

**Status:** 🟡 Partial — items 1–5 approved, items 6–12 TBD. New copy surfaces added by layout redesign: belly band (both sides), compartment cover (top teaser + underside third act), tray recess taglines (11 items), back wall interior, base closing message. Add these to the Copy Document when completing items 6–12.

---

### 2.4 Packaging Spec

**Purpose:** Define the container and presentation — design decisions, reveal sequence, color scheme, narrative structure, copy surface assignments.

**Contents:**
- Container type (tin, box, bag, etc.)
- Interior layout and zone structure
- Lid/closure type
- Label/wrap design requirements
- Interior presentation (how items are arranged)
- Unboxing sequence (what do they see first?)

*Does NOT own measurements, thicknesses, or stack math — see Dimensions Spec for those.*

**Status:** ✅ Complete (v6, layout redesign rewrite March 11, 2026 — supersedes v5. Major changes: 2-zone side-by-side layout, hidden mug compartment, die-cut recesses, 2 booklets, 4-magnet closure + belly band, booking frame as separate removable object.)

---

**Purpose:** Guide for any design work (labels, tattoos, stickers, packaging).

**Contents:**
- Overall visual style
- Color palette
- Typography
- Logo usage (if any)
- Mood/tone references
- Examples of what TO do
- Examples of what NOT to do

**Status:** 🟡 Partial

### 2.6 Exterior Spec

**Purpose:** Define what the outside of the box looks like — surfaces, materials, finishes, hardware, element treatments.

**Contents:**
- Exterior surfaces, materials, and finishes
- Hardware details (clasps, handle, hinges)
- Visual treatment and wear/distress aesthetic
- All exterior surface descriptions

*Does NOT own dimensions — references Dimensions Spec for surface sizes.*

**Status:** ✅ Complete (v3, March 11, 2026 — surface dimensions updated for 14" × 11.5" × 5.0" box. Artboard reference corrected. Element placements and design treatments unchanged.)

### 2.9 Dimensions Spec *(NEW)*

**Purpose:** Single authority for all measurements, thicknesses, stack math, coordinates, and derived dimensions. Every other document references this rather than restating numbers.
